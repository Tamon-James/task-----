# Personal Task & Daily Report Manager レジュメ

## 1. アプリ概要

Personal Task & Daily Report Manager は、Notion の代替として個人のタスク管理、研究進捗管理、学習進捗管理、中長期目標管理、日報作成を一元化するために開発した Web アプリケーションです。

毎日の作業確認から夜の日報作成までを 1 つの画面群で完結できるようにし、PC とスマートフォンの両方から利用できるレスポンシブデザインを採用しています。


## 2. 開発目的

卒業研究、プログラミング学習、個人開発、日々のタスクを別々のツールで管理していたため、情報が分散しやすいという課題がありました。

本アプリでは、以下の流れを Web アプリ内で管理できることを目的としました。

- 朝: 今日のタスク、目標、予定を確認する
- 昼: タスクの進捗やステータスを更新する
- 夜: 日報を入力し、翌日の課題を整理する
- 必要時: ChatGPT に相談しやすい形式の日報を生成する

## 4. 主な機能

### ホームダッシュボード

`index.php`

ログイン後に最初に確認するメイン画面です。

表示内容:

- 今日の日付
- 今日のタスク完了率
- 今日の目標
- 本日のタスク一覧
- 本日のスケジュール
- 未完了タスク
- 今週の目標
- 中長期目標の進捗
- 日報入力への導線

公開環境で、タスク、目標、未完了タスク、完了率が表示されることを確認済みです。

### タスク管理

`tasks.php`

日々の作業を登録・更新するための画面です。

機能:

- タスク作成
- タスク編集
- タスク削除
- タスク検索
- ステータス管理
- プロジェクトとの紐付け

管理項目:

- タスク名
- 詳細
- 所属プロジェクト
- 優先度
- 予定日
- 予定作業時間
- ステータス

ステータス:

- 未着手
- 進行中
- 完了
- 保留

優先度:

- 高
- 中
- 低

### プロジェクト管理

`projects.php`

研究、学習、個人開発などのまとまった活動単位を管理します。

機能:

- プロジェクト作成
- プロジェクト編集
- プロジェクト削除
- プロジェクト一覧表示

管理項目:

- プロジェクト名
- 説明
- 開始日
- 終了予定日
- 状態

状態:

- 進行中
- 完了
- 保留

### 日報管理

`daily_report.php`

1 日の振り返りと翌日の準備を行うための画面です。

入力項目:

- 実施内容
- できなかったこと
- 明日の課題
- 明日使える時間
- 明日すでに入っている予定
- 気づき
- リスク
- ChatGPT に相談したい内容

保存した日報をもとに、ChatGPT に貼り付けやすい形式のテキストを自動生成します。

生成形式例:

```text
2026年6月3日日報

【実施内容】
...

【できなかったこと】
...

【明日の課題】
...

【明日使える時間】
...

【明日すでに入っている予定】
...

【気づき】
...

【リスク】
...

【ChatGPTに相談したい内容】
...
```

生成された日報はワンクリックでコピーできます。

## 5. データベース設計

主なテーブル:

- `projects`
- `tasks`
- `daily_reports`
- `long_goals`
- `schedules`

タスクはプロジェクトに紐付け可能で、プロジェクト削除時にはタスク側の `project_id` を `NULL` にする設計です。

日報は `report_date` をユニークにし、同じ日付の日報は上書き保存できるようにしています。

## 6. セキュリティ対応

### SQLインジェクション対策

DB操作には PDO を使用し、登録・更新・削除・検索処理ではプリペアドステートメントを使用しています。

### XSS対策

画面出力時には `htmlspecialchars` を共通関数 `e()` で使用し、HTMLへの直接出力を避けています。

### ログイン認証

公開環境で個人情報を扱うため、ログイン画面を追加しました。

- セッション管理
- パスワードハッシュによる認証
- ログアウト機能
- 未ログイン時の保護ページリダイレクト

### CSRF対策

フォーム送信には CSRF トークンを埋め込み、POST 処理時に検証しています。

### 設定ファイル保護

DB接続情報やログイン情報は `config.php` に分離し、Git管理対象外にしています。

`.htaccess` では以下を設定しています。

- ディレクトリ一覧表示の禁止
- `config.php` への直接アクセス禁止
- `.sql`、`.log`、`.env` ファイルへの直接アクセス禁止
- `X-Frame-Options`
- `X-Content-Type-Options`
- `Referrer-Policy`

## 7. レスポンシブデザイン

スマートフォン利用を前提に、画面幅 768px 以下で縦並びレイアウトに切り替わるようにしています。

対応内容:

- PCではカードを横並び表示
- スマートフォンではカードを縦並び表示
- 入力欄は幅100%
- 入力フォントは16px以上
- ボタンはタップしやすい高さを確保
- スマートフォンではテーブルをカード形式に変換

## 8. 実装上の工夫

- 共通レイアウトを `includes/layout.php` に分離
- 共通関数を `includes/functions.php` に分離
- 認証・CSRF処理を `includes/auth.php` に集約
- DB接続を `db/db_connect.php` に分離
- CSSとJavaScriptを外部ファイル化
- 初学者でも追いやすいように、1画面1PHPファイルの構成を採用
- 公開環境のドキュメントルートやHTTPS判定の差異を考慮

## 11. アピールポイント

このアプリは、単なるCRUDアプリではなく、毎日の研究・学習・個人開発の進捗管理に実際に使うことを前提に設計しました。

特に、タスク管理と日報管理を分けずに連携させ、夜に ChatGPT へ相談しやすい形式で日報を生成できる点が特徴です。

また、ローカル開発だけでなく CORESERVER 上に公開し、ドメイン設定、HTTPS、ログイン認証、CSS配信、`.htaccess` による保護など、実際の公開運用で発生する問題を解決しながら完成させました。
